# 验收清单

## 内容覆盖

- 列出讲义中的所有例题、练习、巩固题。
- 逐项确认每个题号在 PPT 中存在。
- 特别检查拆分题号：`练2-1`、`练2-2`、`练6-2`、`练6-3`、巩固题等。

## 模板保真

- 题号写在模板原题号框内。
- 题号格式为 `PX-例X`、`PX-练X`、`PX-巩固X`。
- 题号不换行、不溢出，且页码/题型与讲义来源一致；长题号通过拉宽题号框解决，字号保持模板一致。
- 题干位于选项和题图的共同上方；选项在左下，题图在右下，不得让题干与题图并排。
- 题干不得出现无语义硬换行：题干中间被切断、括号单独落行、题干末尾孤立短行都必须回脚本修复。
- 复杂底版必须先确认安全内容区，题干、选项和题图不得贴近边框线、标题条、页脚、logo 或装饰图案。
- 字号、字体、颜色沿用模板实测值。
- 不新增白色底板、卡片或自造题号框。
- 目录页存在且模块名正确。

## 图片

- 题图已放入 PPT。
- 不使用整页 Word/PDF 截图作为主体内容。
- 图片不压题干、不压选项，位置与模板/参考案例一致。
- A/B/C/D 本身是图片的题必须按图片选项题验收：题干在上方，整组选项图在下方可用区域水平居中，不能偏右；不要额外生成空的 A/B/C/D 文本。
- 四图/多图组合选项如需拆图，必须保持原有阅读顺序、等距、等高和对齐关系。
- 白底题图已尽量转为透明底；在深色模板上不应出现突兀白色图片块，除非原图必须保留白底。
- 黑色/深色模板上，题图边框、坐标线、几何线和文字标注应转为白色线稿，不能因透明化后隐入背景。

## 公式

- 逐题对照讲义原式。
- 根号带横线，不是普通勾号字符。
- 上下分式保持上下分式。
- 没有把讲义原式改成等价但形式不同的表达式。
- 选项中的分式、根式、上下标和括号表达式必须使用公式编辑器/OMML 或局部透明公式块，不能以普通文本拼出 `(√2/2)F`、`√(a²+b²)/2`、`kq/r²` 这类代码感表达式。
- 选项文字可编辑；不得把整组选项做成图片。复杂公式、独立公式、选项公式应使用公式编辑器/OMML，局部图片兜底必须有记录。
- 选项公式优先与对应选项文字在同一个文本框/同一段落内以内联公式对象呈现；若公式单独成框，必须逐页检查不与文字、图片或选择框重叠。
- 审查全部公式相关内容：根式、分式、上下标、希腊字母、`kq/r²`、`kQ/(9R²)`、`mgtanθ/q`、`F=kq₁q₂/r²` 等不能作为普通文本留在最终 PPT 中。
- 允许保留为普通文本的只应是中文正文中的单个物理量或短变量，例如“电荷量 q”“夹角 θ”；一旦形成独立表达式，转为公式对象。
- 无 `sqrt`、占位表达式、代码式上标或临时文本残留。

## 程序化检查

运行：

```bash
python3 scripts/audit_pptx.py 输出.pptx --expect-labels labels.txt --fail-layout
```

检查输出中必须确认 `Formula objects` 大于 0，含根号课件必须确认 `radical objects` 大于 0，`raw_sqrt_count` 必须为 0，并且没有 `Formula-like text still in normal text runs`。

版式检查必须关注 `Layout warnings`：不得有对象出界、内容贴边、硬换行、宽图偏心、题干窄框等风险。程序化检查通过后，仍需截图放大检查公式形态、文字重叠、复杂底版边框附近和右上角对象显示。

## 业务反馈专项回归

- 莫名换行：抽查长题干题，确认题干没有被源文件硬换行或窄框挤断。
- 图片选项偏右：抽查 A/B/C/D 为图片的题，确认组合图在选项区居中。
- 复杂底版贴边/不可见：抽查黄条、黑线、logo、页脚等复杂模板页，确认内容不贴边、模板资产不缺失、旧占位对象不残留。
- 公式编辑器：抽查含分式/根式选项题，确认选项公式不是普通文本拼接。
