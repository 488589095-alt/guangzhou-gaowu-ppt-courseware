# 高物讲义转课件

## 核心目标

把高中物理讲义拆解为可授课的 PPT：保留模板风格，保留讲义原题和原公式，题图从讲义内嵌媒体中提取，题干、选项、知识点、目录和总结按模板版式重排。默认不得把 Word/PDF 页面整页截图塞进 PPT。

## 工作流程

1. **读取输入**
   - 模板 PPTX：解析封面、目录、章节页、知识页、例题/练习页、总结页、结束页。
   - 讲义 DOCX/PDF：抽取讲次标题、模块结构、例题、练习、题图、公式和选项。
   - 参考案例：如果用户给了「讲义 + 已生成课件」，先反推页序、题号框、字号、图片位置和取舍规则。

2. **建立内容清单**
   - 形成「讲义页码/位置 → PPT题号 → 类型 → 题干 → 选项 → 图片 → 公式原文」清单。
   - 例题和练习不能漏；对 `练2-1`、`练2-2`、巩固题、补充题等拆分编号要显式列入清单。
   - 生成前先给自己做覆盖检查：讲义中的每个例/练/巩固是否都映射到一页或多页 PPT。

3. **模板保真生成**
   - 复用模板原形状：题号框、正文框、页码、装饰条、背景、目录条目都写回原模板形状或同版式克隆页。
   - 题号格式跟模板：如模板为 `P79-例1`，生成时用 `P7-例4`、`P5-练2-1`，不得改成 `L1-例4`。
   - 字号来自模板实测值。溢出时先拆页，再小幅缩字号；不要把文字放到框外。
   - 模板若是深色/黄黑背景配白字，不得额外加白底板、卡片或新画框破坏统一性。
   - 模板含目录且本讲有多个模块时，默认保留目录页并替换为本讲模块。

4. **图片处理**
   - 从 DOCX/PDF 内嵌媒体或题图区域提取题图，按题目局部放入 PPT。
   - 不得用整页截图代替内容拆解。
   - 题图不能覆盖题干或选项；几何图、受力图、电场图通常靠右或正文下方，跟参考模板一致。

5. **公式处理**
   - 必须保持讲义原式，禁止把讲义中的 `2√3mg/3` 改写成等价的 `2mg/√3`。
   - 讲义是上下分式，PPT也用上下分式；讲义是斜杠式，才保留斜杠式。
   - 根号不能只用普通 `√` 字符。渲染为图片或矢量时，根号必须带横线，如 `√3`、`√(x²+R²)`。
   - 对根式、分式、上下标、希腊字母、复杂选项，优先生成透明公式块，避免不同设备字体渲染不一致。
   - 保留「题号 → 公式源字符串 → 渲染方式」映射，最终反查讲义原文。
   - 详见 [公式渲染规则](references/formula-rendering.md)。

6. **生成后验收**
   - 运行结构反查脚本，检查页数、题号覆盖、风险文本。
   - 生成代表页截图或缩略图，重点看题号框、字号、目录、图片位置、公式形态。
   - 对公式页逐项比对讲义：数学表达式必须原文一致，不能只是“等价”。
   - 详见 [验收清单](references/qa-checklist.md)。

## 推荐检查脚本

使用 `scripts/audit_pptx.py` 反查 PPTX 文本结构：

```bash
python3 scripts/audit_pptx.py 输出.pptx --expect-labels labels.txt
```

`labels.txt` 每行一个预期题号，如：

```text
P4-例1
P5-练2-1
P5-练2-2
P7-例4
```

脚本会检查题号是否缺失，并扫描 `sqrt`、`L1-`、`表达式A/B/C/D`、`...` 等风险文本。复杂公式仍需截图或公式源清单人工复核。

## 完成标准

- 课件页序完整：封面、目录、目标/知识阶梯、章节页、知识页、例练页、总结/挑战/结束页按模板和案例一致。
- 所有例题、练习、巩固题都有对应 PPT 页面。
- 题号框、字号、背景、正文位置与模板一致。
- 题图已放入 PPT，且不是整页讲义截图。
- 公式与讲义原式一致，根号带横线，分式形态正确，无代码式或占位表达式残留。
