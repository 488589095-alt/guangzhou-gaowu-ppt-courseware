# 高物讲义转课件

## 核心目标

把高中物理讲义拆解为可授课的 PPT：保留模板风格，保留讲义原题和原公式，题图从讲义内嵌媒体中提取，题干、选项、知识点、目录和总结按模板版式重排。默认不得把 Word/PDF 页面整页截图塞进 PPT。

## 强制继承 ppt-script-fix 验证闭环

每次生成或修复 PPT 时，同时执行 `ppt-script-fix` 的思路：先定位根因，再改脚本，最后做结构检查和排版质量检查。不能只按用户截图改单页表象。

- **最高级排版红线**：任何页面不得出现文本重叠、文本/图片重叠、文本出框、题目和选项贴近到影响阅读、莫名硬换行。该规则优先于字号、图片大小、单页信息密度和页数控制；若冲突，必须先通过加宽文本框、保留原模板组合框、下移选项/图片、拆页或减少单页内容解决，不能用压缩行距、强行缩图、硬切行来凑。
- **重叠检查必须真实生效**：不能只检查对象是否在画布内。生成后必须估算文字在当前字号、文本框宽度、行距和段前段后下的实际高度，并检查题干、选项、正文、题图之间的有效视觉相交。允许为了减少无意义换行而扩宽文本框，但必须保证实际文字行、题图和边框安全区不相压。
- **少换行优先**：在不出框、不重叠的前提下，题干和选项应尽量保持讲义原句连贯，不得因为脚本估算过保守而把一句话拆成碎行。只有超过可用宽度或为避免重叠/出框时才允许自然换行；换行前必须优先尝试扩大文本框、调整图片区域或拆页。
- **换行是最后手段**：生成和修复脚本必须把 WPS/PowerPoint 文本框内边距、CJK 字符实际宽度和混合中英文宽度纳入估算。若最长选项或题干在安全区内单行可容纳，就要通过清零无效内边距、拉宽文本框、缩小/右移题图来保持单行；不得仅因默认窄框或默认内边距造成莫名换行。
- **紧凑排版优先**：在不重叠、不出框的前提下，页面应尽量紧凑，不得为了“留白好看”插入空段落、空行或过大的段间距。正文、题干和选项只保留语义需要的换段；能放在同一行且不压图时，不主动换行。
- **段落格式同等重要**：拆解模板时，字体/字号/颜色之外，必须记录并复用段落格式，包括 `a:pPr` 的行距 `lnSpc`、段前 `spcBef`、段后 `spcAft`、对齐方式 `algn`、缩进/左右边距 `marL/marR`、项目符号和文本框内边距。题页选项尤其要按模板段落格式生成；广州高物当前模板选项优先保持 1.5 倍行距（`spcPct=150000`）、段前 0，不得额外插入段后空白或空段落来制造间隔。
- **反馈按根因全量修复**：用户给问题文档或截图时，先凝练根因并扫描整套 PPT，不得只修截图页。典型根因包括：题干硬换行、图片选项误判成右侧题图、复杂底版未扣安全区、根式/分式仍是普通文本。每个根因必须应用到所有同类页，并在报告中说明影响范围。
- 视觉问题先写清「问题 → 根因 → 受影响场景 → 修改后可能影响」，再一次性修复同类页面。
- 修完后必须反查题号覆盖、旧内容残留、对象出框、图片/文字重叠风险、公式对象残留、字体/字号/颜色是否与模板实测一致。
- 用户指出的字体、字号、颜色不一致属于高优先级问题；生成后必须程序化抽查 `a:rPr` 的 `typeface`、`sz`、`solidFill`，不能只凭肉眼或文本内容判断。
- 若无法导出缩略图，也必须说明，并用 PPTX 内部坐标、文本框尺寸、图片尺寸和样式属性做替代检查。

## 工作流程

1. **读取输入**
   - 模板 PPTX：解析封面、目录、章节页、知识页、例题/练习页、总结页、结束页。
   - 讲义 DOCX/PDF：抽取讲次标题、模块结构、例题、练习、题图、公式和选项。
   - 参考案例：如果用户给了「讲义 + 已生成课件」，先反推页序、题号框、字号、图片位置和取舍规则。

2. **建立内容清单**
   - 形成「讲义页码/位置 → PPT题号 → 类型 → 题干 → 选项 → 图片 → 公式原文」清单。
   - 例题和练习不能漏；对 `练2-1`、`练2-2`、巩固题、补充题等拆分编号要显式列入清单。
   - 生成前先给自己做覆盖检查：讲义中的每个例/练/巩固是否都映射到一页或多页 PPT。
   - 对每道题标记版式类型：普通文字题、文字+右侧题图题、图片选项题、公式选项题、复杂底版题。不同类型必须使用对应的布局策略，不能统一套右侧小图框或普通文本框。

3. **模板保真生成**
   - 复用模板原形状：题号框、正文框、页码、装饰条、背景、目录条目都写回原模板形状或同版式克隆页。
   - 先验证生成工具的“导入模板→导出PPT”是否保真：封面标题分色、Part页左侧图案、版式层装饰、题号框位置必须与模板一致。若导入导出会丢失图案、改写字体颜色/字号或破坏 run 样式，必须改用 PPTX 底层 OOXML 克隆原模板页，再替换文本和图片；不得继续靠调坐标修补已被破坏的模板。
   - 复用模板页时必须区分「模板背景/装饰/内容容器」和「旧讲次内容」。模板原有背景图形、白色内容框、卡通人物、标题底形、装饰线、分隔线、图标等都属于模板资产，不能因为讲义配图要透明底而删除。只清理旧讲次的文本、旧题图、答案标记、旧公式和无用占位内容。
   - 封面页、Part页等强模板页优先在原文本框内替换 run 内容，保留原坐标、原字号层级、原颜色层级和原版式图案；禁止删除后重画一套“看起来相似”的标题页。
   - 封面标题若模板拆成多个 run（例如数字一个 run、中文标题一个 run），必须分别替换对应 run，保留各自字体、字号、颜色；不要把整句写进第一个 run 造成中文字体回退。
   - 知识页标题和正文必须继承模板的字体和颜色体系；生成后反查 `a:rPr` 的 `typeface`、字号和颜色，不能只看文字内容正确。
   - 题干、选项、知识正文等文本框必须继承模板段落格式；拆模板时要为每类文本框建立「字体 + 字号 + 颜色 + 行距 + 段前段后 + 对齐 + 缩进/边距」样式清单，生成时按角色套用。
   - 知识页、题页、目录页的布局必须先参考模板同页型：如模板 P3 有白色知识框和卡通装饰，生成 P3 必须保留这些模板图形，并将内容写入模板允许的内容区域；不得改成纯色大背景或自造普通文本框。
   - 题号格式跟模板：如模板为 `P79-例1`，生成时用 `P7-例4`、`P5-练2-1`，不得改成 `L1-例4`。
   - 题号必须与讲义来源页和题型一致，不能只按课件顺序猜测；长题号不得换行，且字号必须保持模板一致。若超框，优先拉宽题号框或调整内边距，不得靠缩小字号解决。
- 题干位置参考原讲义和模板：题干必须位于选项和题图的共同上方，选项在左下，题图在右下；不要让题干只占左侧并与题图并排。
- 题页排版必须采用互斥区域：题干上方、选项左侧/下方、题图右侧/下方不得互相覆盖。普通“文字选项 + 右侧题图”页要先为题图保留右栏，再限制选项文本框宽度；A/B/C/D 为图片的题，即使没有 `Options` 文本框，也要根据 `Option Label` + 多张题图识别为图片选项题并整体重排。
- 普通选项页在不压图、不出框时，必须优先拉宽选项文本框以减少无意义换行；不能把短选项放进窄框导致 B/C/D 中途断行。若选项变宽后会压住题图，先缩小、右移或下移题图，再考虑自然换行。选项文本框应按最长选项的单行宽度估算，而不是只按总高度估算。
- 短文字选项配横向组合题图时，选项列和组合图要作为一个整体在可用区内居中/紧凑排布；不得把组合图贴到最右侧，也不得为了避让图片把短选项无意义换行。
   - 黄框/内框型题页若模板样式为 P1 参考页，应采用“题标在左上黄条、题干在白框上方、题图在中部居中、选项在底部”的排版节奏；不能套普通左选项右题图样式。短选项配横向图时，图不得靠右，整组应收紧到阅读中心。
   - 题干和题号不得出现“莫名换行”：抽取 DOCX/PDF 后先清理源文件中的硬换行、断行和 OCR 断句；只有题号、题干、选项之间的语义分隔才能保留换行。生成时题干优先使用横向全宽内容框，禁止把长题干塞入窄框后依赖自动换行。
   - 黄框/内框型宽版模板必须以内框为安全区：题标框、题干、选项和题图都要向内留边，不能贴住左侧黑线、底部花边或黄色标题条；左上角组合题标应整体移动，不能只移动文字或只移动其中一个部件。
   - 题干、正文和选项不得插入空段落。题干和正文中的换行必须有语义；选项只在 A/B/C/D 之间分段，不得在选项内部人为空行。
   - 题干末尾孤立短行、括号单独落行、公式前后被硬切行，都视为生成失败；必须通过加宽文本框、重排题图、拆页或重新分段解决，不能只缩小字号。
   - 字号来自模板实测值。溢出时先拆页，再小幅缩字号；不要把文字放到框外。
   - 用户明确指定字号时必须严格执行，例如题目角标 18、题干 18、选项 20；生成脚本不得再做隐藏缩放或按画布比例缩放字号。
   - 模板若是深色/黄黑背景配白字，不得额外加白底板、卡片或新画框破坏统一性。
   - 模板含目录且本讲有多个模块时，默认保留目录页并替换为本讲模块。
   - 复用模板页时必须删除不用的原始内容图案、答案圈、示意图、占位符等残留；如果残留来自版式层无法删除，应换用无残留的模板底页再重排。
   - 复用知识页、例题页前必须清理旧模板内容的所有对象类型：普通文本框、图片、组合形状、连接线、graphicFrame，以及文本框内残留的 `m:oMath` / `m:oMathPara` 公式对象。不能只删除 `a:t` 文本，因为公式对象可能仍会显示在 PPT 中。

4. **图片处理**
   - 从 DOCX/PDF 内嵌媒体或题图区域提取题图，按题目局部放入 PPT。
   - 不得用整页截图代替内容拆解。
   - “图片透明底”只适用于从讲义提取的题图/配图，不适用于模板自带背景图形。模板自带的白色知识框、白色题目框、装饰图案不是待透明化对象，必须保留。
   - 题图不能覆盖题干或选项；几何图、受力图、电场图通常靠右或正文下方，跟参考模板一致。
   - 题干必须先占据题页上方完整阅读区域；长题干时，题图要随题干高度下移到右下区域，不能插入题干行间或压住题干。
   - 题图必须按原图长宽比等比缩放并居中放入图片框，禁止把图片强行拉满固定框导致变形。
   - 低像素题图不得过度放大；生成脚本应读取原图像素尺寸并设置放大上限，宁可图小一些，也不要糊成大块。
   - 如果 A/B/C/D 选项本身是图片或一整张组合图，必须按“图片选项题”处理：先保留题干在上方，再把整组选项图作为主体内容在可用区域水平居中或按模板选项区居中，不能沿用普通右侧题图策略导致整体偏右。
   - 图片选项题不得额外生成一列空的 A/B/C/D 文本，也不得把组合图裁成导致选项相对位置错乱的碎图；确需拆图时必须保留四个选项的横向/纵向对齐关系。
   - 图片选项题的 A/B/C/D 标签必须继承题干/选项正文主色，不得误用题号绿色或章节色；标签要贴近并对齐对应图片的左侧或左下角，不能留在网格单元边缘造成与图片脱节。
   - 若选项只是 `A. 甲 / B. 乙 / C. 丙 / D. 丁` 这类短文字，而题图是一张横向组合图，也应按“组合题图主体页”处理：左侧保留窄选项列，横向组合图放大并占据主要阅读区域，不能缩成右侧小配图。
   - 横向组合图、四小图合成图、传送带/多状态示意图等宽图需要单独使用横向大图框，不能套普通右侧小图框；生成后检查宽图显示宽度是否接近可用区域。
   - 题图背景应尽量透明化：白底题图转为透明底，避免在深色模板上出现白色图片块；若模板背景为黑色/深色，题图深色线条、边框和文字要同步转为白色线稿，保证可读。
   - 浅色模板也要透明化题图背景，避免出现白色图片矩形块；透明图插入后必须抽查预览确认图片没有压住题干或选项。

5. **公式处理**
   - 必须保持讲义原式，禁止把讲义中的 `2√3mg/3` 改写成等价的 `2mg/√3`。
   - 讲义是上下分式，PPT也用上下分式；讲义是斜杠式，才保留斜杠式。
   - 根号不能只用普通 `√` 字符。渲染为图片或矢量时，根号必须带横线，如 `√3`、`√(x²+R²)`。
   - 选项默认必须可编辑，不得把整组选项渲染成一张图片。根式、分式、上下标、希腊字母、`kq/r²`、`mgtanθ/q`、`F=kq₁q₂/r²` 等独立公式表达式，优先用 PowerPoint 公式对象/OMML/公式编辑器策略处理；只有无法稳定表达的局部公式才可用透明公式块兜底，且不能覆盖整组选项的可编辑性。
   - 选项中的分式、根式和括号表达式不能用普通文本拼接，例如 `(√2/2)F`、`√(a²+b²)/2`、`kq/r²` 必须进入公式编辑器/OMML 或局部透明公式块；最终观感要像公式而不是代码字符串。
   - 选项中的局部公式优先放进对应选项的同一个文本框/同一段落，作为内联公式对象跟随文字排版；不要默认用独立悬浮公式框贴在选项旁边，以免与文字、图片或选择框重叠。
   - 公式对象不只用于“纯公式选项”。题干、正文、小结和中文选项中夹杂的 `mₐ`、`mᵦ`、`F1/F₂`、`Ff = μFN`、`0≤F≤3N` 这类行内公式，也必须拆成同段落内联公式对象，不能因为外面有中文就整句作为普通文本。
   - 公式验收不能只检查根号。生成后必须统计 PPT 内部公式对象数量，并扫描普通文本框中是否仍残留公式候选；除中文正文中的单个变量外，独立公式和选项公式不得作为普通文本留存。
   - 保留「题号 → 公式源字符串 → 渲染方式」映射，最终反查讲义原文。
   - 详见 [公式渲染规则](references/formula-rendering.md)。

6. **生成后验收**
   - 运行结构反查脚本，检查页数、题号覆盖、风险文本。
   - 生成代表页截图或缩略图，重点看题号框、字号、目录、图片位置、公式形态。
   - 必须抽查每类页面的字体、字号、颜色：封面标题、目录/Part 标题、知识页标题、知识页正文、题号、题干、选项、答案/标注。检查方式优先为 PPTX 内部 `a:rPr` 属性反查；发现与模板样页不一致必须回脚本修复。
   - 输出前必须再对照模板样页文本框：逐项比较题号、题干、选项、知识正文的 `a:rPr` 和 `a:pPr`，包括行距、段前段后、对齐、缩进/边距；只检查字号字体不算通过。
   - 必须检查模板资产是否被误删：背景、白色内容框、卡通/图案、标题底形、装饰线、页脚页码、目录图形。若模板资产缺失，不允许交付。
   - 多图选择题必须检查是否重复出现两套 A/B/C/D；如果选项本身只有图片，不要再额外生成一列空的 A/B/C/D 文本。
   - 对业务反馈中暴露的四类问题做专项检查：题干/题号异常换行、图片选项整体偏右、复杂底版内容贴边或对象不可见、选项公式没有使用公式编辑器。
   - 若修复对象是已生成 PPTX，优先运行 `scripts/fix_courseware_layout.py` 做根因后处理：合并题干硬断行、统一选项 1.5 倍行距、把图片选项/宽图放回安全内容区、将根式/分式选项转为公式对象，然后再跑审查脚本。
   - 全页缩略图检查后，还要放大检查首页、P2、知识页、含多图题页和长题干页，确认没有文字/图片重叠。
   - 反查 PPTX 内部坐标：所有文本框和图片必须在画布内；图片显示框比例必须与源图比例一致；题干/选项按字号和行数估算不得超出文本框高度。
   - 反查 PPTX 内部重叠：题干、选项、正文、讲义题图之间不得有有效视觉相交；对 `Options <> rIdXX.png`、`Problem Stem <> rIdXX.png`、`image-image` 等告警必须回脚本修复。若为减少换行而扩大文本框，必须用实际文字高度/行宽与题图安全区复核，而不是仅凭对象外框通过。
   - 反查 PPTX 内部空段落：课程内容中不得出现 `empty_paragraph_in_content`；如果为了避让内容而插入空行，必须改成调整区域、压缩无效间距或拆页。
   - 反查 PPTX 内部公式对象：非公式页不得残留模板原公式；若本讲没有生成公式对象，`m:oMath` / `m:oMathPara` 计数应为 0。发现 `ΔI`、`Δt`、旧讲次公式等模板残留时必须删除后重生成。
   - 对公式页逐项比对讲义：数学表达式必须原文一致，不能只是“等价”。
   - 详见 [验收清单](references/qa-checklist.md)。

## 推荐检查脚本

使用 `scripts/audit_pptx.py` 反查 PPTX 文本结构：

```bash
python3 scripts/audit_pptx.py 输出.pptx --expect-labels labels.txt
```

`labels.txt` 每行一个预期题号，如：

```text
P4-例1
P5-练2-1
P5-练2-2
P7-例4
```

脚本会检查题号是否缺失，并扫描 `sqrt`、`L1-`、`表达式A/B/C/D`、`...` 等风险文本。复杂公式仍需截图或公式源清单人工复核。
脚本还会统计 `m:oMath` 公式对象和根式对象，拦截普通文本框中的公式候选，并报告文本/图片贴边、图片选项偏心、硬换行等版式风险；最终交付建议增加 `--fail-layout`。确需保留普通文本公式时才使用 `--allow-plain-formulas`。
业务反馈专项规则详见 [业务反馈问题固化规则](references/business-feedback-rules.md)。

修复已生成课件时使用：

```bash
python3 scripts/fix_courseware_layout.py 输入.pptx 输出-全量修复版.pptx
```

该脚本只处理生成内容层，不删除模板背景、装饰、组合图案或封面/章节页资产。

## 完成标准

- 课件页序完整：封面、目录、目标/知识阶梯、章节页、知识页、例练页、总结/挑战/结束页按模板和案例一致。
- 所有例题、练习、巩固题都有对应 PPT 页面。
- 题号框、字号、背景、正文位置与模板一致。
- 题图已放入 PPT，且不是整页讲义截图。
- 选项文本可编辑，不能整块图片化。
- 公式与讲义原式一致，根号带横线，分式形态正确，无代码式或占位表达式残留。
